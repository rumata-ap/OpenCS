# CSfea.CScore — мост между CScore и CSfea

Библиотека связывает **поперечные сечения OpenCS** (`CScore.CrossSection`, `CScore.PlateSection`) с **полиморфным откликом сечения** в МКЭ (`CSfea.Core`).

---

## Зачем отдельный проект

| Проект | Роль |
|--------|------|
| `CScore` | Фибровый метод, диаграммы σ–ε, интеграл по сечению (кН, кН·м) |
| `CSfea.Core` | МКЭ: балки, оболочки, решатели (Н, Н·м — СИ) |
| **`CSfea.CScore`** | Адаптеры осей и **единиц** между двумя мирами |

`CSfea.Core` **не зависит** от `CScore`. Мост подключается там, где нужны расчёты ЖБ/стальных сечений из базы OpenCS.

---

## Единицы измерения (обязательно к прочтению)

### Стержневое сечение (`CrossSection`)

| Величина | CScore | CSfea | Переход |
|----------|--------|-------|---------|
| Осевая сила N | кН | Н | × **1000** |
| Момент Mx, My | кН·м | Н·м | × **1000** |
| Жёсткость EA, EI… (∂F/∂ε) | кН, кН·м | Н, Н·м | × **1000** |
| Деформация ε₀, кривизны ky, kz | 1, 1/м | 1, 1/м | **без масштаба** |

Напряжения в `CScore` — **МПа**. Площади — **м²**. Интеграл даёт кН и кН·м автоматически (σ [МПа]·A [м²] = кН).

### Плитное сечение (`PlateSection`, на 1 м ширины)

| Величина | CScore | CSfea | Переход |
|----------|--------|-------|---------|
| Nx, Ny, Nxy | кН/м | Н/м | × **1000** |
| Mx, My, Mxy | кН·м/м | Н·м/м | × **1000** |
| Блоки A, B, D, As | кН/м, кН·м/м | Н/м, Н·м/м | × **1000** |
| Мембранные ε, кривизны κ | — | — | без масштаба |
| Поперечный сдвиг γ_xz, γ_yz | — | — | без масштаба |

Конвертация сосредоточена в `UnitScale.cs` — **не дублируйте ×1000 в прикладном коде**.

## Системы координат и соответствие осей

### CScore — поперечное стержневое сечение

Сечение лежит в плоскости **(x, y)**. Плоскость деформаций (гипотеза плоских сечений):

```
ε(x, y) = ε₀ + ky·y + kz·x
```

| Символ | Смысл |
|--------|--------|
| **x, y** | координаты точки в плоскости сечения [м] |
| **ε₀** (e0) | осевая деформация в начале координат |
| **ky** | кривизна: прирост ε по **y** |
| **kz** | кривизна: прирост ε по **x** |

Результирующие усилия (интеграл по площади):

| Усилие | Формула | Связь с кривизной |
|--------|---------|-------------------|
| **N** | ∫ σ dA | ε₀ |
| **Mx** | ∫ σ·y dA | **ky** |
| **My** | ∫ σ·x dA | **kz** |

**Знак σ:** растяжение σ > 0, сжатие σ < 0 (как в диаграммах материала).

**Знак моментов:** положительный **Mx** — «растяжение в +y» (верх/низ по соглашению контура); **My** — аналогично по **x**. На практике важно **согласовать ориентацию контура** с осями балки (см. ниже).

---

### CSfea — балка (Эйлер–Бернулли / коротационная)

Локальная ось элемента: **x** — вдоль стержня. Поперечное сечение в плоскости **(y, z)**.

| DOF / деформация | Ось | Результирующий момент |
|----------------|-----|------------------------|
| ε₀ | осевая | N |
| **κ_y** (kappaY) | изгиб вокруг **y** | **M_y** |
| **κ_z** (kappaZ) | изгиб вокруг **z** | **M_z** |

2D-рама (узлы в плоскости xy): 3 DOF на узел `[u, v, θ_z]`. Изгиб в плоскости — момент **M_z**, кривизна **κ_z**.

---

### Таблица соответствия CScore ↔ CSfea (балка)

Эталон — Python `CrossSectionBeamResponse` в GreenSectionPy.

| CSfea (балка) | CScore (сечение) | Примечание |
|---------------|------------------|------------|
| ε₀ | e0 | напрямую |
| **κ_y** | **ky** | |
| **κ_z** | **kz** | |
| **M_y** | **Mx** | перестановка индексов |
| **M_z** | **My** | перестановка индексов |
| N | N | та же ось, разные единицы |

**Оси сечения на чертеже (CScore):**

```
CScore-x  ↔  ось z балки (CSfea)
CScore-y  ↔  ось y балки (CSfea)
```

Тогда при изгибе в плоскости xy (момент M_z балки) меняется ε вдоль **CScore-y** → используется **ky** → **Mx** в CScore → **M_y** в 3D; в **2D-элементе** задействована связка **κ_z ↔ kz ↔ My ↔ M_z**.

> **Практическое правило:** контур сечения в OpenCS ориентируйте так же, как в Python-эталоне: горизонталь чертежа = **z балки**, вертикаль = **y балки**. Иначе изгиб и кручение «переедут» на другую ось.

Кручение в MVP — **линейное**: `GJ` задаётся отдельно (`TorsionalStiffness`), не из фибрового интеграла.

---

### CSfea — оболочка (MITC3 / MITC4)

Локальная система элемента: **x, y** — в плоскости элемента, **z** — нормаль.

| Группа | Компоненты | Смысл |
|--------|------------|--------|
| Мембрана ε_m | ε_xx, ε_yy, γ_xy | деформации срединной поверхности |
| Кривизны κ | κ_xx, κ_yy, κ_xy | изгиб/кручение плиты |
| Сдвиг γ | γ_xz, γ_yz | поперечный сдвиг (Миндлин) |

`PlateSection` в CScore использует ту же кинематику `ShellStrainState`:

```
ε_x(z) = ε₀x + κ_x·z ,   ε_y(z) = ε₀y + κ_y·z ,   γ_xy(z) = γ₀xy + κ_xy·z
```

**Оси плиты в CScore совпадают с локальными осями оболочечного элемента CSfea** (x, y в плоскости, z — толщина). Перестановки, как у балки, **нет** — только масштаб ×1000.

Поперечный сдвиг: **Q = As · γ** (линейный As, по умолчанию k_shear·G·h).

## Правила знаков в МКЭ CSfea

### Балка 2D

Узел i → j вдоль локальной **x**. DOF: `(u_i, v_i, θ_i, u_j, v_j, θ_j)`.

- **N > 0** — растяжение стержня (конец j отходит от i).
- **v > 0** — перемещение по локальной **y** (поперечь).
- **θ_z > 0** — поворот узла против часовой стрелки (из +x в +y).
- **M_z > 0** — изгиб «вогнутость в +y» (сжатие на +y-волокнах) — согласовано с положительной θ у консоли под поперечной нагрузкой в +y.

### Балка 3D

6 DOF: `(u, v, w, θ_x, θ_y, θ_z)`. Моменты `M_y`, `M_z` — вокруг локальных y и z по правилу правого винта.

### Оболочка

- **Nx, Ny > 0** — растяжение по соответствующей оси в плоскости элемента.
- **Mx, My > 0** — изгиб: положительный момент увеличивает кривизну κ_xx / κ_yy (зависит от направления нормали z элемента).
- Нормаль **z** строится по геометрии элемента (`ShellGeometry.LocalFrame`); при сомнениях проверяйте знак на простом прогибе под симметричной нагрузкой.

## API

```csharp
using CSfea.CScoreBridge;
using CScore;

// Балка: сечение уже подготовлено
section.ResolveAndBuildDiagramms();
var beamResp = SectionBridgeFactory.BeamFromPrepared(section, CalcType.C, gjLinear: G * J);

// Балка: из справочника материалов
var beamResp2 = SectionBridgeFactory.BeamFromDatabase(section, materialsById, CalcType.C);

// Плита
var shellResp = SectionBridgeFactory.ShellFromDatabase(plateSection, materialsById, CalcType.C);

// МКЭ
var mesh = new FrameMesh2D(nodes, elems, beamResp);
var u = mesh.SolveLinear(f, fixedDofs);
```

## Типичные ошибки

1. **Забыли ×1000** — прогиб в 1000 раз не тот (если обошли мост и подали кН в CSfea напрямую).
2. **Перепутали Mx/My** — контур повёрнут на 90° относительно балки; изгиб идёт не в той плоскости.
3. **Не вызвали `ResolveAndBuildDiagramms()`** — пустые диаграммы, нулевые усилия.
4. **Разные CalcType** в задаче и в мосте — неверная жёсткость длительной/кратковременной схемы.
