# Одноосная диаграмма N-M для OpenSees

## Цель

Добавить в существующий OpenSees backend OpenCS расчёт одноосной диаграммы взаимодействия `N-M`. Первая версия получает упорядоченный список продольных сил и для каждой силы выполняет отдельный monotonic moment–curvature расчёт уже реализованным stage 0–1 backend. Результат должен быть доступен как библиотечный тип и как задача OpenCS.

## Границы первой версии

В scope входят:

- `N` в единицах backend SI (`N`), а в `ParamsJson` задачи — в `kN`;
- выбранная существующая ось `Mx` или `My`;
- общие для всех точек `maxCurvature`, `increments`, executable и timeout;
- последовательный запуск точек с отдельным каталогом артефактов для каждой точки;
- сохранение последней сошедшейся строки `M`, достигнутой кривизны, статуса, диагностики и пути артефактов;
- итоговые статусы `ok`, `not_converged`, `error`;
- отмена между точками и передача `CancellationToken` во внутренний анализ.

Порядок сил из запроса сохраняется; значения должны быть конечными, список — непустым, дубликаты запрещены. Точка считается имеющей значение диаграммы, если внутренний анализ вернул хотя бы одну сошедшуюся строку; используется последняя такая строка. Если все точки завершились успешно, итоговый статус `ok`. При отсутствии ошибок, но наличии несошедшихся точек — `not_converged`; ошибки подготовки/запуска дают `error`.

## Архитектура

В `OpenCS.OpenSees.Analysis` добавляются `SectionInteractionRequest`, `SectionInteractionPoint` и `SectionInteractionResult`. Запрос содержит `IReadOnlyList<double> AxialForcesN`, параметры кривизны, ось и coordinate convention. Точка содержит исходный `N`, последнюю сошедшуюся `SectionHistoryRow?`, статус, диагностику и каталог артефактов.

`SectionAnalysisService` реализует новый небольшой контракт `ISectionAnalysisExecutor`. `SectionInteractionService` зависит от этого контракта и последовательно создаёт `SectionAnalysisRequest` для каждой силы, не дублируя Tcl, runner, parser или artifact store. Каждый вызов существующего сервиса получает один и тот же executable/timeout, но создаёт уникальный каталог через `OpenSeesArtifactStore`.

В OpenCS добавляются `OpenSeesSectionInteractionParams` и `OpenSeesSectionInteractionHandler` с kind `opensees_section_interaction_nm`. Handler переиспользует CScore adapter, конвертирует силы `kN -> N`, запускает interaction service и сериализует типизированный результат в `CalcResult.DataJson`. В оба словаря ресурсов добавляется название задачи.

## Обработка ошибок и артефактов

Сбой отдельной точки не удаляет её каталог и не скрывает stdout/stderr, Tcl, manifest и exit metadata. Последующие точки не запускаются после ошибки подготовки модели; ошибка/несходимость конкретного внешнего анализа фиксируется в соответствующей точке. При отмене новые точки не запускаются, уже созданные артефакты сохраняются.

## Тестирование

Добавляются unit-тесты валидации запроса, выбора последней сошедшейся строки, сохранения порядка `N`, агрегации статусов и уникальности каталогов. Отдельный task-contract тест проверяет `ParamsJson`, конвертацию `kN -> N` и сериализацию результата. Существующие 41 тест stage 0–1 и реальные opt-in integration-тесты должны продолжить проходить. `N-Mx-My`, target-force пары, параллельные запуски и WPF-график остаются отдельными следующими срезами.
