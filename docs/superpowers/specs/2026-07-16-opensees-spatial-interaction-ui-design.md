# UI пространственной диаграммы N-Mx-My OpenSees: дизайн

## Цель

Добавить пользовательский путь для пространственной OpenSees-задачи в стиле существующего `CalcTaskPropsDialog` и специализированных result views OpenCS.

Пользователь выбирает сечение, тип расчёта и существующий `ForceSet`. Значения `N` берутся из всех строк выбранного набора, а `Mx` и `My` строк набора игнорируются: OpenSees сам строит полный оборот направлений изгиба для каждого `N`.

## Task dialog

Новый kind:

```text
opensees_section_interaction_n_mx_my
```

Он появляется в общем списке `CalcTaskPropsDialog` рядом с другими OpenSees-задачами. Для него используются стандартные поля сечения, набора усилий и `CalcType`; выбор отдельной строки `LoadItem` скрывается, потому что task работает со всем `ForceSet`.

В специальной группе параметров отображаются:

- выбранный `ForceSet` и предварительный список уникальных `N` в кН;
- `angleStepDegrees`, по умолчанию `45`;
- `maxCurvature`, по умолчанию `0.01 1/м`;
- `increments`, по умолчанию `20`;
- `timeoutSeconds`, по умолчанию `300`;
- необязательный `executablePath`.

Preview показывает количество направлений `360 / angleStepDegrees` и количество точек `N × directions`. Поля принимают русский и invariant decimal separator так же, как существующие поля диалога. До сохранения проверяются выбранное сечение, выбранный bar `ForceSet`, непустой набор конечных уникальных `N`, положительный шаг угла, положительные `maxCurvature`, `increments` и `timeout`.

`CalcTask.ForceSetId` хранит выбранный набор. `ForceItemId` равен `0`. `ParamsJson` содержит:

```json
{
  "angleStepDegrees": 45,
  "maxCurvature": 0.01,
  "increments": 20,
  "timeoutSeconds": 300,
  "executablePath": "C:/path/to/OpenSees.exe"
}
```

При повторном открытии task диалог восстанавливает выбранный набор и параметры из `ParamsJson`; старые параметры не теряются при изменении только UI.

Все видимые подписи, единицы, подсказки, ошибки и название task добавляются в `Strings.ru-RU.xaml` и `Strings.en-US.xaml`. В XAML и code-behind не добавляются пользовательские строки напрямую.

## Result view

`CalcResultView` маршрутизирует новый kind в `OpenSeesSpatialInteractionResultView`. View не изменяет исходный JSON и не запускает повторный расчёт.

Основная вкладка `Mx–My` содержит:

- ComboBox выбранной продольной силы `N` из результата;
- квадратный график с осями `Mx`, `My` в кН·м и замкнутым полярным контуром точек по углам;
- выделенную точку выбранного угла;
- таблицу направлений с `θ`, `Mx`, `My`, `κMx`, `κMy`, статусом и ссылкой/кнопкой открытия каталога артефактов;
- сводный статус задачи и диагностику.

Вкладка `История M–κ` включается по желанию пользователя. Она использует выбранные `N` и `θ` и показывает радиальную историю из одной точки, не подменяя полярный контур. Если в результате нет `TerminalRow`/истории или точка имеет ошибку, вкладка показывает диагностическое состояние вместо пустого графика.

Вкладка `Диагностика` показывает агрегированные сообщения и список каталогов точек. Абсолютные пути не выводятся как исполняемые команды; они отображаются как текст и доступны для открытия только через существующий безопасный file-dialog/service-паттерн.

Для графиков используется существующая инфраструктура ScottPlot/WPF или текущий локальный `PlotCanvas`, если он уже применяется в конкретном result view. Новая визуализация не добавляет 3D-рендерер и не требует внешнего UI-пакета. Масштабирование должно сохранять одинаковый масштаб по Mx и My, чтобы форма контура не искажалась.

## Модель представления

`OpenSeesSpatialInteractionResultVM` десериализует `SectionSpatialInteractionResult`, группирует точки по `AxialForceN`, сортирует группы только по порядку результата и предоставляет:

- `AvailableAxialForces`, `SelectedAxialForce`;
- `AvailableAngles`, `SelectedAngle`;
- `PolarSeries` и выделенную точку;
- `HistorySeries` для выбранной точки;
- форматированные значения и локализованные статусы;
- команды выбора точки, открытия артефактов и подгонки графика.

При смене `SelectedAxialForce` выбранный угол сохраняется, если такой угол есть в новой группе; иначе выбирается первая точка группы. При смене `SelectedAngle` обновляются выделение, карточка точки и вкладка `M–κ`.

## Тестирование UI-контрактов

- тест parser-а параметров диалога проверяет сохранение/восстановление `ForceSetId` на уровне `CalcTask` и defaults JSON;
- тесты VM проверяют группировку по `N`, выбор угла, пустые/ошибочные точки и преобразование Н→кН, Н·м→кН·м;
- contract-тест проверяет регистрацию kind и маршрутизацию `CalcResultView`;
- ручная проверка WPF подтверждает: выбор ForceSet, preview числа `N`, редактирование существующей task, полярный контур, выбор строки таблицы, вкладку `M–κ`, диагностику и отсутствие хардкодных видимых строк.

## Не входит

В эту UI-версию не входят редактирование списка `N` вручную, ручное редактирование углов, 3D-поверхность, интерактивное изменение расчётных точек и параллельный запуск.
